# UX Guidelines

## Principe

Le parent doit sentir qu'il cree un moment doux, pas qu'il remplit un dossier.

## Formulaire

- Maximum une decision principale par etape.
- Montrer la progression.
- Rendre les champs optionnels vraiment optionnels.
- Toujours afficher un recapitulatif avant generation.
- Ne pas demander de photo d'enfant en V1.

## Mobile-first

Le parcours doit etre confortable sur mobile :

- boutons pleine largeur quand necessaire ;
- cartes theme faciles a toucher ;
- champs espacés ;
- recapitulatif lisible ;
- navigation sans menus complexes.

## Rassurance

Ajouter des micro-textes :

- "Vous pourrez verifier avant de generer."
- "Aucune photo n'est necessaire."
- "L'histoire reste un support de lecture, pas un avis medical."

## Resultat

La page histoire doit etre calme et immersive :

- couverture en haut ;
- resume court ;
- pages separees comme un livre ;
- morale claire ;
- conseils de lecture pour le parent ;
- actions futures visibles mais non frustrantes.

## Erreurs

Les erreurs doivent etre precises :

- "Ajoutez le prenom de l'enfant pour personnaliser l'histoire."
- "Choisissez un theme avant de continuer."
- "Ajoutez un objectif en quelques mots."
