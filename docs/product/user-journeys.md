# Parcours Utilisateurs

## Creation rapide depuis la landing

1. Le parent arrive sur la landing.
2. Il comprend que StoryKid cree une histoire personnalisee pour une emotion ou une etape.
3. Il clique sur "Creer une histoire".
4. Il choisit un theme.
5. Il renseigne quelques details simples.
6. Il valide l'aperçu.
7. Il consulte l'histoire demo.
8. Il voit les options futures : PDF, audio, livre imprime.

## Consultation pricing

1. Le parent lit un exemple ou genere une histoire.
2. Il veut savoir ce qui est inclus.
3. Il consulte `/pricing`.
4. Il compare digital, audio et livre imprime.
5. Il identifie l'offre adaptee.

## Retour dashboard

1. Le parent se connectera plus tard via Supabase Auth.
2. Il retrouve ses histoires.
3. Il ouvre une histoire existante.
4. Il pourra exporter, modifier ou commander.

## Points de friction a surveiller

- Trop de champs dans le formulaire.
- Manque de confiance dans la confidentialite.
- Ambiguite entre support educatif et promesse therapeutique.
- Pricing trop precoce avant perception de qualite.
